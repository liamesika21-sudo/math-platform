1. הצגת קבצים — cat, more, less

טקסט הוא הממשק העיקרי של מערכות UNIX/Linux. הפקודות הבאות מציגות תוכן של קבצי טקסט:

פקודה תיאור
cat file מציגה את כל תוכן הקובץ
more file מציגה מסך אחד בכל פעם — גלילה למטה בלבד
less file כמו more אבל ניתן לגלול למעלה ולמטה ולחפש

2. צפייה בחלקי קובץ — head ו-tail

head — תחילת הקובץ
head [options] [files]

ברירת מחדל: 10 שורות ראשונות.
  • -nX — X שורות ראשונות
  • -n-X — הכל חוץ מ-X האחרונות
  • -cX — X בתים ראשונים
  • -c-X — הכל חוץ מ-X בתים אחרונים
tail — סוף הקובץ
tail [options] [files]

ברירת מחדל: 10 שורות אחרונות.
  • -nX — X שורות אחרונות
  • -n+X — משורה X ואילך
  • -cX — X בתים אחרונים
  • -c+X — מבית X ואילך
  • -f — מעקב חי אחרי קובץ (log)
דוגמאות (על הקובץ phone_dir.txt)
head -n3 phone_dir.txt
ADAMS, Andrew 7583 BARRETT, Bruce 6466 BAYES, Ryan 6585
tail -n3 phone_dir.txt
NAKAMURA, Satoshi 6453 REEVE, Shirley 7391 ROSNER, David 6830
tail -n+12 phone_dir.txt # from line 12 onwards
MEASDAY, David 6494 NAKAMURA, Satoshi 6453 REEVE, Shirley 7391 ROSNER, David 6830

3. ספירה — wc (Word Count)

הפקודה wc
wc [options] [files] — סופרת שורות, מילים ובתים בקובץ.
  • -l — מספר שורות
  • -w — מספר מילים
  • -c — מספר בתים
ללא דגלים — מציגה את שלושתם.

4. הפנית קלט/פלט — Redirection

כל פקודה ב-Linux מקבלת קלט מ-stdin (מקלדת) ושולחת פלט ל-stdout ול-stderr (מסך).

אופרטור תפקיד דוגמה
> או 1> הפניית stdout לקובץ (דריסה) ls > files
2> הפניית stderr לקובץ ls *.c 2> err
&> הפניית stdout + stderr לקובץ ls *.c &> all
< הפניית קובץ כ-stdin לפקודה wc < file.txt
דוגמה: הפרדת stdout ו-stderr
ls -l *.txt *.tmp 1>out 2>err # stdout to 'out', stderr to 'err' ls -l *.txt *.tmp 1>out 2>>err # stderr appended to 'err'
הערה
הפניית קלט עם < אינה נפוצה — רוב הפקודות מקבלות קובץ כארגומנט, ולשאר ניתן להשתמש ב-pipes.

5. הוספה לקובץ — Append

ההבדל בין > ל->>
  • > — דורס את הקובץ (יוצר חדש אם לא קיים)
  • >> — מוסיף לסוף הקובץ (append) ללא דריסה
דוגמה
echo first line > lines.txt echo second line >> lines.txt echo third line >> lines.txt cat lines.txt
first line second line third line
echo fourth line > lines.txt # overwrites! cat lines.txt
fourth line

6. צינורות — Pipes

מהו Pipe?
Pipe (|) לוקח את ה-stdout של פקודה אחת ומעביר אותו כ-stdin של פקודה אחרת.

command1 [args] | command2 [args]
למה זה חשוב?
  • ניתן לשרשר (chain) מספר פקודות — ליצור כלים מורכבים מאבני בניין פשוטות.
  • אין צורך בקבצים זמניים — הנתונים זורמים ישירות.
  • סדרת pipes נקראת לפעמים filters.
  • Pipes הם סוג פשוט של inter-process communication (IPC).
דוגמאות
# Count .txt files in current directory ls *.txt | wc -w
# Print lines 3-4 of phone_dir.txt head -n4 phone_dir.txt | tail -n2
BAYES, Ryan 6585 BECK, Bill 6346
# Same result, different approach tail -n+3 phone_dir.txt | head -n2
# Most recently changed file ls -t | head -n1

7. חיתוך שדות — cut

הפקודה cut
cut [options] [files] — מדפיסה תת-קבוצה של תווים/שדות מכל שורה.
  • -c<RANGE> — חיתוך לפי תווים (לדוגמה: -c3-4,6,9-)
  • -f<RANGE> — חיתוך לפי שדות (לדוגמה: -f3-4,6,9-)
  • -d<C> — הגדרת תו מפריד (delimiter). ברירת מחדל: tab
RANGE מוגדר עם מקפים (-) ופסיקים (,).
דוגמאות
# First 2 first names (space as delimiter, field 2) cut -f2 -d" " phone_dir.txt | head -n2
Andrew Bruce
# Last 3 last names (comma as delimiter, field 1) cut -f1 -d"," phone_dir.txt | tail -n3
NAKAMURA REEVE ROSNER

8. הדבקת קבצים — paste

הפקודה paste
paste [options] [files] — שמה קבצים זה לצד זה (side by side), שורה מול שורה.
  • -d<C> — תו מפריד בין הקבצים. ברירת מחדל: tab
דוגמאות
# Paste phone_dir.txt alongside zipcodes, separated by tab paste -d"\t" phone_dir.txt zipcodes | head -n3
ADAMS, Andrew 7583 34980 BARRETT, Bruce 6466 12567 BAYES, Ryan 6585 87492
# Repeat each line 3 times, separated by colons paste -d"::" zipcodes zipcodes zipcodes
34980:34980:34980 12567:12567:12567 ...

9. מיון — sort

הפקודה sort
sort [options] [files] — ממיינת שורות לפי סדר לקסיקוגרפי (אלפביתי) כברירת מחדל.
  • -r — סדר הפוך
  • -n — מיון מספרי (9 לפני 21, במקום 21 לפני 9)
  • -k<N>,<M> — מיון לפי מפתח מהמילה ה-N עד סוף המילה ה-M
ברירת מחדל: מפריד = whitespace (רווח, tab).
הבדל חשוב: -k2,2 מול -k2
  • -k2,2 — ממיין לפי שדה 2 בלבד. שוויונות נשמרות בסדר המקורי.
  • -k2 — ממיין לפי שדה 2 ועד סוף השורה (שדות 2, 3, ... מאוחדים למחרוזת אחת).
מפתחות מיון ראשיים ומשניים
# Primary: field 2 numerically, Secondary: field 3 numerically sort -k2,2n -k3,3n fruits.txt
date 1 10 banana 1 20 apple 2 10 cherry 2 5 fig 3 7
# Get first names + phone numbers, sort by name then by number cut -f2-3 -d" " phone_dir.txt | sort -k1,1 -k2,2n

10. הסרת כפילויות — uniq

הפקודה uniq
uniq [options] [file] — מאחדת שורות עוקבות זהות לשורה אחת.
  • -c — מוסיפה ספירה (count) לפני כל שורה
שימו לב: uniq מסירה רק כפילויות רצופות! לכן כמעט תמיד משלבים עם sort קודם.
דוגמאות
cat repeat.txt uniq repeat.txt sort repeat.txt | uniq
first line first line first line first line second line second line second line first line third line first line third line third line
sort repeat.txt | uniq -c
3 first line 1 second line 1 third line

11. החלפת תווים — tr (Translate)

הפקודה tr
tr [options] set1 [set2] — מחליפה כל תו מ-set1 בתו המתאים מ-set2.

חשוב: קוראת מ-stdin בלבד (אין ארגומנט קובץ). יש להשתמש עם pipe או <.

טווחים ומחלקות תווים

תחביר משמעות
"a-z"כל האותיות הקטנות
"A-Z"כל האותיות הגדולות
"DEFa-z"D, E, F וכל אותיות a עד z
[:alpha:]כל האותיות
[:upper:]אותיות גדולות בלבד
[:lower:]אותיות קטנות בלבד
[:digit:]כל הספרות
[:space:]רווח, tab, newline
[:punct:]סימני פיסוק

דגלים

-s (squeeze)
מאחד חזרות רצופות של תו לתו אחד.
echo 'AB*****CD' | tr -s "*"
AB*CD
-d (delete)
מוחק את התווים שב-set1 (בלי set2).
echo 'AABabCD' | tr -d "Ab"
BaCD
דוגמאות
# Convert all uppercase to lowercase cat phone_dir.txt | tr [:upper:] [:lower:]
adams, andrew 7583 barrett, bruce 6466 ...
# Swap case (upper↔lower) cat phone_dir.txt | tr "[:upper:][:lower:]" "[:lower:][:upper:]"
adams, aNDREW 7583 barrett, bRUCE 6466 ...
# Delete all uppercase letters echo 'AABabCD' | tr -d [:upper:]
ab

12. דוגמאות משולבות — File Processing with Pipes

Task I — רשימת שמות פרטיים ייחודיים

# Step 1: cut second column cut -f2 -d" " phone_dir.txt # Step 2: sort cut -f2 -d" " phone_dir.txt | sort # Step 3: remove duplicates cut -f2 -d" " phone_dir.txt | sort | uniq
Abe Andrew Bill ...

Task II — כמה שמות מתחילים בכל אות?

# Steps 1-2: cut first char of first name cut -f2 -d" " phone_dir.txt | cut -c1 # Steps 3-4: sort and count unique cut -f2 -d" " phone_dir.txt | cut -c1 | sort | uniq -c
2 A 2 B 2 D 1 J ...

Task III — ספר טלפונים "שגוי" עם מספרים מעורבבים

# Step 1: cut 3rd column, sort numerically descending, save to file cut -f3 -d" " phone_dir.txt | sort -rn > nums.txt # Step 2: paste alongside original, separated by dash paste -d"-" phone_dir.txt nums.txt
ADAMS, Andrew 7583-9749 BARRETT, Bruce 6466-7583 BAYES, Ryan 6585-7484 ...

נקודות עיקריות

  • הצגת קבצים: cat, more, less, head, tail.
  • ספירה: wc -l (שורות), wc -w (מילים), wc -c (בתים).
  • Redirection: > (stdout), 2> (stderr), &> (שניהם), < (stdin). >> = append.
  • Pipes (|): מעבירים stdout של פקודה אחת ל-stdin של הבאה. ניתן לשרשר.
  • Text delimited tables: cut (חיתוך שדות/תווים), paste (הדבקה side-by-side), sort (מיון).
  • uniq: הסרת שורות עוקבות זהות. -c לספירה. חייב sort לפני!
  • tr: החלפת/מחיקת/איחוד תווים. קורא מ-stdin בלבד. דגלים: -s (squeeze), -d (delete).
טיפ לשיעורי בית
  • השתמשו בנתיבים יחסיים (relative paths).
  • השתמשו רק בפקודות/דגלים שנלמדו בהרצאות ובתרגולים.
  • ודאו שהפתרון עובד על השרת (בעיות hidden characters ממערכות הפעלה שונות).
