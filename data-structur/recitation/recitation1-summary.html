1. ADT vs Implementation — Linked List

ADT: Linked List
Objects: סדרה סופית של List Elements.

Operations on the list:
  • Create list — יצירת רשימה חדשה
  • Get first element — קבלת האיבר הראשון
  • Insert new element as first — הכנסת איבר חדש כראשון
  • Delete first element — מחיקת האיבר הראשון

Operations given a list element:
  • Get/Set data
  • Get/Set next element
List Element
A List Element is an object containing both data and a handle (pointer/reference) to another List Element.

Fields:
  • data — name, image, etc.
  • next — handle to the next element

2. מימוש Linked List — Node Objects

מימוש באמצעות Node Objects
כל node object מכיל:
  • Value (data) — הנתון עצמו (זיכרון מוקצה)
  • Next (handle/pointer) — מצביע ל-node הבא

\(L \rightarrow \text{head node} \rightarrow \text{node} \rightarrow \ldots \rightarrow \text{NULL}\)
סימונים בפסאודו-קוד
  • \(\text{head}[L]\) = האיבר הראשון ברשימה
  • \(\text{data}[x]\) = הנתון של איבר \(x\)
  • \(\text{next}[x]\) = האיבר הבא אחרי \(x\)

3. מימוש Linked List — מערכים (Data + Handles)

למה מערכים?
  • שימוש בפוינטרים אמיתיים קשה או לא נתמך בשפות מסוימות (Fortran, Basic)
  • לפעמים יעיל יותר כאשר מספר האיברים ידוע מראש
מבנה L
  • L.D[] — מערך נתונים (data array)
  • L.N[] — מערך אינדקסים (integer/index array for next pointers)
  • L.n — גודל המערך
  • L.head — אינדקס האיבר הראשון ברשימת הנתונים
  • L.free — אינדקס האיבר הראשון ברשימת המקומות הפנויים (free list)
מיפוי פוינטרים
Pointer הוא אינדקס שלם. null = -1.
p.data = L.D[p],   p.next = L.N[p]
אתחול (Initialization)
  • \(\text{L.head} = \text{null} = -1\)
  • \(\text{L.free} = n-1\)
  • מערך N מאותחל כ-free list מקושרת הפוכה:
\(N[0]=-1, \quad N[1]=0, \quad N[2]=1, \quad \ldots, \quad N[n-1]=n-2\)

כלומר: שרשרת הפוכה — \(n{-}1 \rightarrow n{-}2 \rightarrow \ldots \rightarrow 1 \rightarrow 0 \rightarrow \text{null}\)

4. פסאודו-קוד: InsertFront ו-DeleteFront

InsertFront(L, x)
if L.free ≠ -1 then q ← L.free L.free ← L.N[L.free] L.D[q] ← x L.N[q] ← L.head L.head ← q else return "overflow"
DeleteFront(L)
if L.head = -1 then return "underflow" else q ← L.head L.head ← L.N[q] L.N[q] ← L.free L.free ← q
דוגמה מעשית: L.n=8, L.head=3 (data: a→b→c), L.free=6

InsertFront(L, x):

q = 6 → L.free = 4 → L.D[6] = x → L.N[6] = 3 → L.head = 6

DeleteFront(L):

q = 3 (was 'a') → L.head = 5 → L.N[3] = 6 → L.free = 3
סיבוכיות
שתי הפעולות InsertFront ו-DeleteFront הן \(O(1)\).
כאשר \(\text{L.free} = -1\) (אין תאים פנויים), יש להכפיל את גודל המערך.
סיבוכיות מימוש מערכים
פעולהBest CaseWorst Case
Insertion\(O(1)\)\(O(n)\)
Deletion\(O(1)\)\(O(n)\)

5. הוכחה באינדוקציה — מבנה ודוגמאות

מבנה הוכחה באינדוקציה
טענה: \(S(n)\) נכונה לכל \(n \geq c\).

  1. Base case (בסיס): הוכח ש-\(S(c)\) נכונה.
  2. Inductive Hypothesis (הנחת אינדוקציה): נניח ש-\(S(k)\) נכונה לכל \(k < n\).
  3. Inductive Step (צעד אינדוקציה): הוכח ש-\(S(n)\) נכונה.

הערה: לעיתים קרובות מניחים רק \(k = n - 1\).
דוגמה: הוכח כי \(\displaystyle\sum_{i=1}^{n} i = \frac{n(n+1)}{2}\) לכל \(n \geq 1\)

Base: \(n = 1\)

\(\sum_{i=1}^{1} i = 1 = \frac{1 \cdot 2}{2} \checkmark\)

IH: נניח

\(\sum_{i=1}^{k} i = \frac{k(k+1)}{2}\)

IS: נוכיח עבור \(k+1\)

\(\sum_{i=1}^{k+1} i = \sum_{i=1}^{k} i + (k+1) = \frac{k(k+1)}{2} + (k+1) = \frac{k(k+1) + 2(k+1)}{2} = \frac{(k+1)(k+2)}{2} \checkmark\)

6. "כל הסוסים באותו צבע" — למה ההוכחה נכשלת

ה"הוכחה" (שגויה)
Base case \(n=1\): סוס אחד = צבע אחד — טריוויאלי.

IH: כל קבוצה של \(n\) סוסים היא באותו צבע.

IS: בקבוצה של \(n+1\) סוסים:
  • הורד את האחרון → \(n\) סוסים באותו צבע (לפי IH)
  • הורד את הראשון → \(n\) סוסים באותו צבע (לפי IH)
  • החפיפה (overlap) בין שתי הקבוצות מקשרת את הצבעים → כל \(n+1\) באותו צבע
למה ההוכחה נכשלת?
הטיעון דורש חפיפה (overlap) בין שתי הקבוצות של \(n\) סוסים.

במעבר מ-\(n=1\) ל-\(n=2\): הורדת הראשון והאחרון מקבוצה של 2 סוסים נותנת שתי קבוצות יחידניות (singletons):
\(\{h_1\} \quad \text{and} \quad \{h_2\}\)
אין חפיפה! הקבוצות זרות (disjoint), אז אין קשר בין הצבעים.

האינדוקציה נכשלת ספציפית במעבר מ-\(n=1\) ל-\(n=2\).

7. סדרה חשבונית (Arithmetic Sequence)

הגדרה
סדרה חשבונית: ההפרש בין איברים עוקבים קבוע \(d\).
\(a_n = a_1 + (n-1)d\)
\(a_n = a_m + (n-m)d\)
סכום סדרה חשבונית
\(S_n = \frac{n}{2}(a_1 + a_n) = \frac{n}{2}[2a_1 + (n-1)d]\)
דוגמה
\(2 + 5 + 8 + 11 + 14 = \frac{5(2 + 14)}{2} = \frac{5 \cdot 16}{2} = 40\)

8. סדרה הנדסית (Geometric Sequence)

הגדרה
סדרה הנדסית: כל איבר = הקודם כפול יחס קבוע \(r\).
\(a_n = a \cdot r^{n-1}, \qquad a_n = a_{n-1} \cdot r\)
סכום סופי
\(S_n = \frac{a(1 - r^n)}{1 - r} \quad \text{or} \quad \frac{a(r^n - 1)}{r - 1} \quad \text{when } r > 1\)
דוגמה
\(2 + 10 + 50 + 250 = \frac{2(1 - 5^4)}{1 - 5} = \frac{2(1 - 625)}{-4} = \frac{2 \cdot (-624)}{-4} = 312\)
סכום אינסופי (\(0 < r < 1\))
\(S_\infty = \frac{a}{1 - r}\)
דוגמה:
\(\frac{1}{2} + \frac{1}{4} + \frac{1}{8} + \ldots = \frac{1/2}{1 - 1/2} = 1\)

9. חזקות — זהויות שימושיות

זהויות חזקות
\(a^0 = 1\)
\(a^1 = a\)
\(a^{-1} = \frac{1}{a}\)
\(a^{m+n} = a^m \cdot a^n\)
\((a^m)^n = a^{mn} = (a^n)^m\)
\((b \cdot c)^n = b^n \cdot c^n\)

10. לוגריתמים — זהויות והמרת בסיס

הגדרה
\(\log_b(x) = y \quad \Longleftrightarrow \quad b^y = x\)
דוגמה: \(\log_2(64) = 6\) כי \(64 = 2^6\).
זהויות מרכזיות
  • \(a = b^{\log_b a}\)
  • \(\log_c(ab) = \log_c a + \log_c b\)
  • \(\log_b(a^n) = n \cdot \log_b a\)
  • \(\log_b a = \dfrac{\log_c a}{\log_c b}\)
  • \(\log_b\!\left(\dfrac{1}{a}\right) = -\log_b a\)
  • \(\log_b a = \dfrac{1}{\log_a b}\)
  • \(a^{\log_b c} = c^{\log_b a}\)
הוכחה: \(\log_a(bc) = \log_a b + \log_a c\)
לפי הגדרה:
\(b = a^{\log_a b}, \qquad c = a^{\log_a c}\)
לכן:
\(bc = a^{\log_a b} \cdot a^{\log_a c} = a^{\log_a b + \log_a c}\)
אבל גם:
\(bc = a^{\log_a(bc)}\)
מהשוואת המעריכים:
\(\log_a(bc) = \log_a b + \log_a c \quad \checkmark\)
המרת בסיס (Base Change)
\(\log_b x = \frac{\log_k x}{\log_k b}\)
שינוי הבסיס משנה את הערך בגורם קבוע בלבד.

11. Floor ו-Ceiling

הגדרות
  • Floor \(\lfloor x \rfloor\) = המספר השלם הגדול ביותר ש-\(\leq x\)
  • Ceiling \(\lceil x \rceil\) = המספר השלם הקטן ביותר ש-\(\geq x\)
דוגמאות
ביטויתוצאה
\(\lfloor -2.7 \rfloor\)\(-3\)
\(\lfloor 2.7 \rfloor\)\(2\)
\(\lfloor 2 \rfloor\)\(2\)
\(\lceil -2.3 \rceil\)\(-2\)
\(\lceil 2.3 \rceil\)\(3\)
\(\lceil 2 \rceil\)\(2\)

12. דוגמת אינדוקציה עם רקורסיה ולוגריתמים

שאלה
נגדיר: \(a_1 = 0\), ולכל \(n \geq 2\): \(a_n = a_{\lfloor n/2 \rfloor} + 1\).

הוכח: אם \(n = 2^k\) אז \(a_n = \log_2(n) = k\).
פתרון — אינדוקציה על \(k\)

Base: \(k = 1\)

\(n = 2, \quad a_2 = a_{\lfloor 2/2 \rfloor} + 1 = a_1 + 1 = 0 + 1 = 1 = \log_2(2) \checkmark\)

IH: נניח עבור \(k \geq 1\)

\(a_{2^k} = k\)

IS: נוכיח עבור \(k+1\)

\(a_{2^{k+1}} = a_{\lfloor 2^{k+1}/2 \rfloor} + 1 = a_{2^k} + 1 = k + 1 = \log_2(2^{k+1}) \checkmark\)

13. תרגיל פסאודו-קוד — Find-Max

שאלה
כתוב פסאודו-קוד עבור Find-Max על רשימה מקושרת: מקבל רשימה מקושרת של מספרים, מחזיר את המקסימום.
פתרון
Find-Max(list)
curr = list.head curr_max = -∞ while curr ≠ null curr_max = Maximum(curr_max, curr.data) curr = curr.next return curr_max
מהו פסאודו-קוד טוב?
רשימת בדיקה:
  • האם הוא חד-משמעי (unambiguous)?
  • האם אדם יכול להבין אותו ללא תלות בשפת תכנות?
  • האם מישהו יכול להפוך אותו לתוכנית?

אם "כן" לכולם → פסאודו-קוד טוב.
אל תלחצו בגלל תחביר — כל עוד זה ברור, זה נכון.

14. סיכום

ADT → Implementation → Induction → Series → Logarithms → Pseudocode
  • ADT (Linked List): הפרדה בין הגדרת הממשק למימוש. Objects + Operations.
  • מימוש ב-Node Objects: כל node מכיל data ו-next pointer. סימון: \(\text{head}[L]\), \(\text{data}[x]\), \(\text{next}[x]\).
  • מימוש במערכים: שני מערכים D[] ו-N[] + ניהול free list. Pointers הם אינדקסים, null = -1.
  • InsertFront / DeleteFront: שתיהן \(O(1)\). כשאין מקום — מכפילים מערך \(O(n)\).
  • אינדוקציה: Base case + הנחה + צעד. זהירות מחפיפה (overlap) — דוגמת הסוסים.
  • סדרה חשבונית: \(S_n = \frac{n}{2}(a_1 + a_n)\). סדרה הנדסית: \(S_n = \frac{a(1-r^n)}{1-r}\).
  • חזקות ולוגריתמים: זהויות מרכזיות. המרת בסיס משנה בגורם קבוע.
  • Floor/Ceiling: \(\lfloor x \rfloor\) = שלם גדול ביותר \(\leq x\), \(\lceil x \rceil\) = שלם קטן ביותר \(\geq x\).
  • פסאודו-קוד: חד-משמעי, עצמאי משפה, ניתן למימוש. תחביר לא קריטי — בהירות כן.
