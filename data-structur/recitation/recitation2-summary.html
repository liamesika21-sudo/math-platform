1. חזרה: ניתוח Amortized

תזכורת
Amortized analysis ממצע את עלות הפעולות על פני סדרה שלמה, ו"מחליק" פעולות יקרות נדירות.
דוגמה: Array Doubling
  • Best case: \(O(1)\) — הכנסה רגילה ללא resizing
  • Worst case: \(O(n)\) — resizing והעתקת כל המערך
  • אבל resizing קורה רק כשמספר האיברים הוא חזקה של 2
  • Amortized Cost = \(O(1)\)

2. מונה בינארי — ניתוח Amortized

מונה בינארי
מונה בינארי סופר מ-\(0\) עד \(n-1\) בייצוג בסיס 2, באמצעות מערך של \(k = \lceil\log n\rceil\) ביטים.
דוגמה: ספירה מ-0 עד 7 (מערך בגודל 3)
מספרייצוג בינארי
0[0,0,0]
1[0,0,1]
2[0,1,0]
3[0,1,1]
4[1,0,0]
5[1,0,1]
6[1,1,0]
7[1,1,1]

כמה flips סה"כ?

  • ה-bit הכי פחות משמעותי (LSB) מתהפך כל צעד — \(n\) פעמים.
  • ה-bit השני מתהפך כל 2 צעדים — \(\lfloor n/2 \rfloor\) פעמים.
  • ה-bit ה-\(i\) מתהפך כל \(2^{i-1}\) צעדים — \(\lfloor n/2^i \rfloor\) פעמים.
\(\text{Total flips} = n + \lfloor n/2 \rfloor + \lfloor n/4 \rfloor + \ldots + \lfloor n/2^k \rfloor\)
חישוב
\(n + \lfloor n/2 \rfloor + \lfloor n/4 \rfloor + \ldots \leq n + \frac{n}{2} + \frac{n}{4} + \ldots + 1 = n\left(1 + \frac{1}{2} + \frac{1}{4} + \ldots\right) \leq n \cdot 2 = 2n\)

לכן ה-amortized cost של צעד בודד = \(\frac{2n}{n} = 2 = O(1)\).

3. Binary Heap — הגדרה ותכונות

הגדרה: Binary Heap
Heap הוא מבנה נתונים שתומך בפעולות Priority Queue.

Binary Heap הוא עץ בינארי כמעט-שלם (almost-complete) עם תכונת הערמה (heap property).
Almost-Complete Binary Tree
  • כל הרמות חוץ מהאחרונה מלאות לגמרי
  • ברמה האחרונה — העלים ממלאים משמאל לימין
תכונת הערמה
  • Max-Heap: כל צומת \(\geq\) כל צאצאיו
  • Min-Heap: כל צומת \(\leq\) כל צאצאיו
(בתרגול עובדים עם max-heap)

4. מימוש Heap במערך

כיוון ש-heap הוא עץ כמעט-שלם, ניתן לאחסן אותו במערך בלי פוינטרים:

ניווט במערך (אינדקס מתחיל מ-1)
פעולהנוסחה
הורה של צומת \(i\)\(\text{Parent}(i) = \lfloor i/2 \rfloor\)
ילד שמאלי של \(i\)\(\text{Left}(i) = 2i\)
ילד ימני של \(i\)\(\text{Right}(i) = 2i + 1\)
דוגמה: Max-Heap במערך

העץ:

16 → [14, 10] → [8, 7, 9, 3] → [2, 4, 1]

המערך:

\([16, 14, 10, 8, 7, 9, 3, 2, 4, 1]\)

לדוגמה: צומת 2 (ערך 14), ילדיו: \(2 \times 2 = 4\) (ערך 8) ו-\(2 \times 2 + 1 = 5\) (ערך 7).

5. Percolate Up / Down

שתי פעולות היסוד לתיקון תכונת הערמה:

PercolateUp(i) — "בועה למעלה"
מזיז צומת למעלה בעץ כל עוד הוא גדול מהוריו.

שימוש: אחרי Insert (הוספה בסוף).
PercolateDown(i) — "שקיעה למטה"
מזיז צומת למטה בעץ כל עוד הוא קטן מאחד הילדים.

שימוש: אחרי DeleteMax (הצבת האחרון בשורש).
פסאודו-קוד
PercolateDown(i): [max-heap] while A[i] is smaller than one of its children: j ← index of maximal child swap A[i] and A[j] i ← j PercolateUp(i): [max-heap] while A[i] is bigger than its parent: j ← index of parent swap A[i] and A[j] i ← j
סיבוכיות
שתי הפעולות מבקרות לכל היותר צומת אחד בכל רמה — לכן \(O(\log n)\).
דוגמה: PercolateDown מהשורש

עץ עם שורש 8, ילדיו 25 ו-13:

8 בשורש → 8 < 25 → swap → 8 < 20 → swap → 8 < 17 → swap → 8 הגיע לעלה

בכל שלב — מחליפים עם הילד הגדול ביותר.

6. פעולות על Max-Heap

פעולהתיאורסיבוכיות
find-max() מחזירים את השורש (המקסימום תמיד בשורש) \(O(1)\)
insert(x, k) מוסיפים כאיבר אחרון → PercolateUp \(O(\log n)\)
delete-max() שומרים שורש, מציבים איבר אחרון בשורש, מוחקים אחרון → PercolateDown \(O(\log n)\)

דוגמה: Insert(43) ל-Max-Heap

העץ לפני: שורש=44, ילדים 42, 35
43 נכנס כעלה אחרון → 43 > 31 (הורה) → swap → 43 > 42 (הורה) → swap → 43 < 44 (שורש) → עצור

תוצאה: 44 בשורש, 43 כילד שמאלי, 42 ירד להיות ילד של 43.

דוגמה: ExtractMax פעמיים

ExtractMax ראשון (מוציא 44)
שומרים 44 → 14 (אחרון) → שורש → 14 < 42 → swap עם max ילד → 14 < 33 → swap → 14 < 26 → swap

תוצאה: [42, 33, 35, 26, 31, 19, 27, 10, 14]

7. BuildHeap

בניית Heap ממערך
ניתן לבנות heap ממערך לא ממוין ע"י PercolateDown מלמטה למעלה — מתחילים מהצמתים הפנימיים האחרונים (הורי העלים) ועולים לשורש.
דוגמה: BuildHeap על [35, 31, 19, 10, 26, 44]
[35,31,19,10,26,44] → PercolateDown(3): 19<44 → swap → PercolateDown(2): 31 OK → PercolateDown(1): 35<44 → swap → [44,31,35,10,26,19]
סיבוכיות BuildHeap

נראה כאילו \(O(n \log n)\) (כי \(n/2\) פעמים PercolateDown), אבל בפועל:

\(\sum_{h=0}^{\log n} \frac{n}{2^{h+1}} \cdot O(h) = O(n)\)

כי רוב הצמתים נמצאים קרוב לתחתית ועושים מעט עבודה. BuildHeap = \(O(n)\).

שאלה 1 — מספר העלים בעץ Almost-Complete

שאלה
הוכיחו: בכל עץ בינארי כמעט-שלם עם \(n\) צמתים, בדיוק \(\lceil n/2 \rceil\) מהצמתים הם עלים.
פתרון

נניח שעומק העץ הוא \(k\), ומספר העלים ברמה \(k\) הוא \(l\) (לא ידוע).

  • מספר הצמתים ברמה \(k-1\): \(2^{k-1}\)
  • מתוך צמתי רמה \(k-1\), \(\lceil l/2 \rceil\) הם הורים של עלי רמה \(k\), והשאר (\(2^{k-1} - \lceil l/2 \rceil\)) הם גם עלים
  • סה"כ עלים = \(l + (2^{k-1} - \lceil l/2 \rceil)\)
  • סה"כ צמתים: \(n = 2^k - 1 + l\) (עץ שלם עד רמה \(k-1\) ועוד \(l\) עלים)

המקרה \(l\) זוגי (\(n\) אי-זוגי):

\(\text{עלים} = l + 2^{k-1} - \frac{l}{2} = 2^{k-1} + \frac{l}{2}\)
\(\left\lceil\frac{n}{2}\right\rceil = \left\lceil\frac{2^k - 1 + l}{2}\right\rceil = \left\lceil 2^{k-1} + \frac{l-1}{2}\right\rceil = 2^{k-1} + \frac{l}{2} \quad \checkmark\)

המקרה \(l\) אי-זוגי (\(n\) זוגי):

\(\text{עלים} = l + 2^{k-1} - \lceil l/2 \rceil = l + 2^{k-1} - \frac{l+1}{2} = 2^{k-1} + \frac{l-1}{2}\)
\(\left\lceil\frac{n}{2}\right\rceil = \frac{n}{2} = \frac{2^k - 1 + l}{2} = 2^{k-1} + \frac{l-1}{2} \quad \checkmark\)

שאלה 2 — מציאת k האיברים הגדולים ביותר ב-Heap

שאלה 2.1 — האיבר השני בגודלו
מציאת המקסימום ב-heap טריוויאלית (השורש). איך מוצאים את השני בגודלו?
פתרון 2.1
השני בגודלו חייב להיות ילד ישיר של השורש (כי הוא הפסיד רק לשורש).
השוו בין שני ילדי השורש — \(O(1)\).
שאלה 2.2 — האיבר השלישי בגודלו
איך מוצאים את השלישי בגודלו?
פתרון 2.2
השלישי בגודלו יכול להיות:
  • ילד של השורש (שאינו השני בגודלו), או
  • ילד של האיבר השני בגודלו
סה"כ 3 מועמדים — השוו ביניהם. \(O(1)\).
שאלה 2.3 — k האיברים הגדולים ביותר
בהינתן max-heap עם \(n\) איברים, מצאו את \(k\) האיברים הגדולים ביותר בסיבוכיות \(O(k \log k)\).
פתרון 2.3 — אלגוריתם "Child Collection"

הרעיון: מנהלים אוסף מועמדים (child collection) — בכל שלב, המועמד הגדול ביותר הוא הבא ב-top-k, וילדיו נכנסים לאוסף.

האלגוריתם:

  1. אתחל את האוסף עם מצביע לשורש
  2. חזור \(k-1\) פעמים:
    • מצא את האיבר עם המפתח הגדול ביותר באוסף
    • הסר אותו מהאוסף
    • הוסף את שני ילדיו לאוסף
  3. החזר את המקסימום באוסף הנוכחי
דוגמה: Heap = [44, 42, 35, 33, 31, 19, 27, 10, 26, 14]
שלבאוסף (אינדקסים)נבחר (ערך)
התחלה{0}44
1{1, 2}42
2{2, 3, 4}35
3{3, 4, 5, 6}33

סיבוכיות

  • גודל האוסף: מתחיל ב-1, גדל ב-1 בכל איטרציה (מוציאים 1, מכניסים 2) → בסוף \(k\) איטרציות: לכל היותר \(k\) איברים
  • אם מממשים את האוסף כ-max-heap עזר: כל פעולת Insert/DeleteMax = \(O(\log k)\)
  • סה"כ: \(k\) איטרציות \(\times\) \(O(\log k)\) = \(O(k \log k)\)
נקודה חשובה
האלגוריתם לא משנה את ה-heap המקורי — הוא רק קורא ממנו. ה-heap העזר הוא מבנה נפרד שמנהל את המועמדים.

שאלה 3 — מימוש Queue עם שני Stacks

שאלה
הציעו דרך לממש Queue (FIFO) באמצעות שני Stacks בלבד (LIFO).
מותר להשתמש רק בפעולות Push, Pop, Top, IsEmpty.
תזכורת
  • Queue (FIFO): Enqueue (הכנסה לסוף), Dequeue (הוצאה מההתחלה)
  • Stack (LIFO): Push (הכנסה), Pop (הוצאת האחרון)
פתרון

משתמשים בשני stacks: S1 (להכנסות) ו-S2 (להוצאות).

Enqueue(x):

Push(S1, x)

פשוט דוחפים ל-S1. \(O(1)\).

Dequeue():

if S2 is empty: while S1 is not empty: Push(S2, Pop(S1)) return Pop(S2)

אם S2 ריק — מעבירים הכל מ-S1 ל-S2 (ההיפוך הופך LIFO ל-FIFO). אחר כך Pop מ-S2.

דוגמה מלאה צעד-אחר-צעד
פעולהS1 (top בימין)S2 (top בימין)תוצאה
Enqueue(5)[5][]—
Enqueue(2)[5, 2][]—
Enqueue(3)[5, 2, 3][]—
Dequeue()[][3, 2]5 (העברה + Pop)
Enqueue(6)[6][3, 2]—
Enqueue(4)[6, 4][3, 2]—
Dequeue()[6, 4][3]2 (S2 לא ריק, Pop)
Dequeue()[6, 4][]3 (Pop)
Dequeue()[][4]6 (העברה + Pop)
ניתוח סיבוכיות
  • Enqueue: \(O(1)\) — תמיד
  • Dequeue worst-case: \(O(n)\) — כשצריך להעביר את כל S1 ל-S2
  • Dequeue amortized: \(O(1)\) — כל איבר מועבר מ-S1 ל-S2 לכל היותר פעם אחת לאורך כל חיי המבנה

נקודות עיקריות

  • מונה בינארי: amortized cost של increment = \(O(1)\) (סה"כ \(2n\) flips ל-\(n\) צעדים).
  • Binary Heap: עץ almost-complete + heap property. מימוש במערך: Parent=\(\lfloor i/2\rfloor\), Left=\(2i\), Right=\(2i+1\).
  • Percolate Up/Down: שתיהן \(O(\log n)\). Up להכנסה, Down למחיקת מקסימום.
  • עלים בעץ almost-complete: בדיוק \(\lceil n/2 \rceil\).
  • Top-k: אלגוריתם Child Collection עם heap עזר — \(O(k \log k)\) בלי לשנות את ה-heap המקורי.
  • Queue מ-2 Stacks: Enqueue = \(O(1)\), Dequeue = \(O(1)\) amortized. כל איבר עובר מ-S1 ל-S2 לכל היותר פעם אחת.
